## Method (and why this one)

icb-platform and `GRP-Costing-System` share **no git history** ŌĆö icb-platform is a fresh repo that imported the calculator *as-is* at `d37716a` *"feat(backend): import Cost Calculator as-is"* (WO v4.12 steps 2-3), all on **2026-06-02**. So drift cannot be a cross-repo `git diff` from a merge-base (there is none), and GitHub's compare view can't align two history-less trees either. Instead:

1. **Pinned the fork instant** ŌĆö icb-platform's earliest commits are all 2026-06-02 (`19c5e1a` scaffold ŌåÆ `d37716a` import).
2. **Listed legacy commits strictly after the fork** ŌåÆ exactly two (`d2da5bf`, `14e6817`, both 2026-06-08).
3. **Content divergence scan** ŌĆö for every legacy `app/**.{py,js,html}`, summed the +/ŌłÆ lines against its icb-platform `backend/app/**` counterpart (`git diff --no-index --numstat`). This catches any drift the dates might miss and separates 7-Jun drift from v4.12 restructuring.

The scan corroborates the commit list precisely: every file whose divergence exceeds the v4.12 baseline is one of the two 7-Jun commits' files, and its line-deficit in icb-platform matches that commit's additions (e.g. `calculator.js ŌłÆ542` Ōåö `d2da5bf +557`; `calculator.py ŌłÆ186` Ōåö `+202`; `results.html ŌłÆ30` Ōåö `+33`). Path mapping is uniform: **legacy `app/X` ŌåÆ icb-platform `backend/app/X`.**

Reproduce:
```bash
# fork point
git -C icb-platform log --reverse --format="%cd %h %s" --date=short main | head -3
# post-fork legacy commits
git -C "Costing model" log --since="2026-06-03" --format="%cd %h %s" --date=short
# divergence scan
LEG="Costing model"; cd icb-platform
git -C "$LEG" ls-files 'app/' | grep -E '\.(py|js|html)$' | while read lf; do
  rel="${lf#app/}"; I="backend/app/$rel"; L="$LEG/$lf"
  if [ -f "$I" ]; then s=$(git diff --no-index --numstat "$L" "$I" 2>/dev/null | awk '{s+=$1+$2} END{print s+0}');
    [ "${s:-0}" -gt 0 ] && printf "%6s  %s\n" "$s" "$rel"; else echo "MISSING $rel"; fi
done | sort -rn
```
