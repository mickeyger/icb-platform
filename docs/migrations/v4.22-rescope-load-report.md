## MRP structure (resolved at build-time)

The `Material Requirement Planning` sheet is **header row 5, data row 6+, 1 363 items**, with the
demand side as **5 day-blocks** (offsets 10/20/30/40/50; each = ITEM CODE, MATERIAL, REQ QTY, JOB 1–6):

- **Row 1** = day headers (DAY 1–5) with the day's date (cols 11/21/31/41/51) → `need_by` + `week_bucket`.
- **Row 3** = the **real 5-digit job numbers** per JOB column (e.g. 32300, 32685, 32498) — above the labels.
- **Row 5** = positional "JOB 1…JOB 6" labels (the v4.21 numeric-header detection does **not** port here).

Each non-zero (item × job-col) cell → a `demand_line`. Result: **390 lines across 15 active job-columns,
12 distinct MRP jobs — all 12 matched to `production_jobs`** by job number (every demand line linked).
The WO's ~3–5K estimate was high: the MRP cells are sparse (most are 0). `sap_code` is stored as-is.

## Findings (for Simeon / v4.23)

1. **⚠️ SAP coding systems are disjoint — 0 / 276 (0%) match.** Across the MRP sheet **276 distinct
   ITEM CODEs** (`GRP-MPS-A-*`, `CON-ALU-A-*`, `STE-PLA-A-*`, …) were seen and **none** match
   `icb_costings.sap_item_codes` (descriptive: `1.2M_KICK_PLATES`). The 390 loaded `demand_lines` span
   **76 distinct sap_codes — all 76 orphaned**. Same disjoint-scheme finding as v4.21. **v4.23 lands
   `icb_sap.OITM` + the `demand_lines.sap_code → OITM.ItemCode` FK (NOT VALID initially)** to reconcile.
2. **Chassis sheet: 321 real records, not ~1 038.** `JOBS & CHASSIS` has `max_row=1038` (allocated
   extent), but only **321 rows hold data** (rows 2–322, contiguous; all with a JOB NO, 254 with a VIN);
   the rest is empty padding. The WO's ~1 038 was the padding-inflated extent. Single-row header (row 1,
   stripped of leading/trailing whitespace); 240 of 254 date cells parsed (DD/MM/YYYY); `vehicle_id_no`
   sometimes holds non-VIN text like "CANCELLED" (stored as-is). Full 112-col row preserved in `raw_row_json`.
3. **Live Daily Count is non-uniform:** header row 3 (ALU/TIMBER/PU) vs 4 (STEEL/EPS/COILS); ALU/STEEL
   carry `REJECTED STOCK`, EPS/COILS lack `PRICE`; the sheets also carry a duplicate `ORDERED` + a SAP
   comparison block (`IN STOCK`/`ORDERED`/`Variance Value`). The loader maps **by header name, first-wins**,
   auto-detecting the header row — so position drift across sheets is handled.
