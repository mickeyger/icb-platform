## Foreign key ΓÇõ DEFERRED (build-time decision; see ADR 0013)

WO ┬º0.5 specified adding `demand_lines.sap_code ΓÁÈ icb_sap."OITM"."ItemCode"` as a **NOT VALID** FK.
During the build the enforced FK proved operationally harmful and is **deferred** (a Q-Ph2D-05
single-blocker call); its **intent (measuring the relationship) is delivered above**. Reasons:

1. **CASCADE nukes demand** ΓÇõ `TRUNCATE icb_sap."OITM" CASCADE` (the inventory reload) cascade-truncated
   `icb_mes.demand_lines` (observed live: demand dropped 390 ΓÁÈ 0; restored after removing the FK).
2. **Blocks the independent reload cycle** ΓÇõ demand (workbook ETL) and OITM (inventory ETL) are
   separate one-shot loaders with no guaranteed ordering; the FK couples them.
3. **Breaks CI seed** ΓÇõ `seed_from_mockup` inserts demo `demand_lines` whose codes aren't in the
   (empty-in-CI) OITM, so the FK would reject the seed.

The **0/76 orphan result proves the FK is satisfiable** ΓÇõ it should be added (and `VALIDATE`d) in a
follow-on micro-WO once the two ETLs are coordinated (OITM before demand) and the demo seed codes
reconciled. **Flagged for BA.**

## API rewire (`/api/materials` + cycle-count baseline)

- `services/materials._materials_select()` now LEFT-joins **`icb_sap.OITW`** (by
  `mes_materials.sap_code = OITW.ItemCode`) instead of `icb_mes.stock_positions`. The `StockPosition`
  **response shape is unchanged**: `OnHandΓÁÈsap_stock`, `IsCommitedΓÁÈallocated`, `AvailableΓÁÈfree`,
  `OnOrderΓÁÈopen_po_qty`, `open_po_etaΓÁÈnull`, `last_refreshedΓÁÈOITW load time`.
- `services/stock_counts.record_count()` reads the cycle-count baseline from `OITW.OnHand`
  (summed across warehouses) ΓÇõ a single coherent SAP-stock source.
- **`free` is now SAP's `Available`** (includes on-order). `icb_mes.stock_positions` is **deprecated**
  (retained for rollback; nothing reads it). CI/dev get OITW data via `seed_from_mockup` mirroring the
  12 mock items into `OWHS`/`OITM`/`OITW`.
