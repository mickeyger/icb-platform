## Data-quality findings (flag to Simeon)

1. **⚠️ SAP coding systems are disjoint — 0 / 5158 (0%) cross-link.** The workbook's MATERIAL PLANNING uses **procurement codes** (`BUY-GEN-A-0095`, `CON-ALU-A-0002`, …); the Cost Calculator's `sap_item_codes` use **descriptive codes** (`1.2M_KICK_PLATES`, `1/2"/BALL_VALVE`, …). They do **not** reconcile by code. Demand is stored on the workbook's own codes (`demand_lines.sap_code`, per §0.5 "as-is"); cross-referencing workbook demand to the costing catalogue would require a **code-mapping table** (future WO).
2. **MATERIAL PLANNING tracks a different job batch than JOBS PLANNED.** Only **2 of the 15** material-planning job columns are in the active PLANNED set → 13 job columns (`32078, 32136, 32137, 32205, 32239, 32240, 32241, 32243, 32244, 32245, …`) have **no `production_jobs` match**, so those demand lines keep `job_ref` (business key) with `production_job_id = NULL`. The material-planning batch is a current vacuum-planning group, largely disjoint from the in-process PLANNED jobs.
3. **Customer-name drift** (see above) — 56% of workbook customer names don't exactly match the migrated customer master.

## planning_slots — MVP derivation (§0.4)

Each non-completed job with a VACUUM date becomes one slot in its **Monday-of-VACUUM-week**, `lane='vacuum'`, `bay='Bay-N'` (position within that week), `status='scheduled'`. This populates a correct, navigable board for Phase 2D testing but **does not mirror ICB's physical bay layout** — the full **336-column PLANNING grid** is explicitly deferred to its own WO (re-scope per Q-Ph2D-05). The board's default 4-week window shows a subset of the 132 slots; the rest are reachable by week navigation.
