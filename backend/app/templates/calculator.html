⚠ Multiple Server Processes on Port 8000

If you're seeing stale pages, 404 errors on new routes, or missing features — multiple uvicorn processes may be competing on port 8000. This happens when old Claude worktree sessions leave their server running.

Step 1 — find all PIDs on port 8000

netstat -ano | findstr ":8000"

Look at the last column — any row marked LISTENING with a unique PID is a competing server.

Step 2 — kill each extra PID

taskkill /PID <pid> /F

Repeat for each stray PID. Keep only the one you started from the root project.

Step 3 — restart cleanly

Open a new terminal at C:\Users\micge\Documents\Costing model\ and run start.bat. Do not run start.bat from inside any .claude\worktrees\* subdirectory.
